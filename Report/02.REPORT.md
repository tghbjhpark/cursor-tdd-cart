## 1. 요약

본 세션은 **MomTest 방식 고객 발견**과 **테스트 가능 계약(Contract) 도출**에 집중했다. 30대 1인 가구 직장인 페르소나 식사 인터뷰를 시작으로, 비즈니스 분석·밀키트 구독 제품 계약·장바구니 할인 도메인 계약표까지 단계적으로 산출했다. **코드 변경·TDD RED/GREEN 사이클은 수행하지 않았다** (`src/cart.py` 비어 있음, `tests/` 미작성).

## 2. 핵심 결정·산출물

### 핵심 결정

- 인터뷰 답변은 **과거 실제 행동·금액·시간**만 계약 근거로 사용; 의도·미래 약속·가짜 칭찬은 제외.
- 계약 ID 체계: `INV-*`(불변식), `E-*`(에러), `AC-*`(수용 조건), `OOS-*`(구현 금지).
- 근거 레벨 L0~L3 적용; **근거 없는 동작은 ID 부여·구현 금지**.
- `subtotal == 50_000` 경계·소수 반올림·쿠폰·배송비는 **OOS**로 보류.

### 장바구니 할인 — 확정 계약 요약

| ID | 핵심 내용 |
|----|-----------|
| INV-1 | `subtotal == Σ(price × qty)` |
| INV-2 | `subtotal > 50_000` → `round(subtotal × 0.9)` |
| INV-3 | `subtotal ≤ 50_000` → 할인 없음 |
| INV-4~6 | VIP: 문턱 10% 후 추가 5%, 순서 고정 |
| E-1~E-4 | 음수 qty/price, None, 빈 장바구니 → 예외 |
| E-5~E-6 | qty=-1 시 0원 표시 금지; 빈 제출 500 금지 |
| AC-1~4 | #1107, #1042, 2024-03-12 시나리오 |
| OOS-3 | `== 50_000` 포함 여부 미확인 |
